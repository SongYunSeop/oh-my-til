Rust Ownership — Interactive Learning
Core Concept
Every value in Rust has exactly one owner.
When the owner goes out of scope, the value is automatically dropped.
Example
let s1 = String::from("hello");
let s2 = s1; // s1 is moved to s2
// println!("{}", s1); // ERROR: s1 no longer valid
Quick Check
After assigning s1 to s2, what happens if you use s1?
a) Prints the same value
b) ✔ Compile error (move semantics)
c) Runtime null reference error
d) Creates a shallow copy
Correct!
Rust uses move semantics. let s2 = s1 transfers ownership,
so s1 is no longer valid. Use .clone() to explicitly copy the value.
