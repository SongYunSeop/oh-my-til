# Raw Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a raw sources layer so that original reference materials are preserved and linked to TILs.

**Architecture:** Skills-only approach. Update 4 skill files + 1 template to support `raw/{category}/{slug}.md` as immutable source storage. No MCP tool changes.

**Tech Stack:** Markdown skill prompts, YAML frontmatter conventions

**Spec:** `docs/superpowers/specs/2026-04-13-raw-layer-design.md`

---

### Task 1: Update /research skill — add raw source saving

**Files:**
- Modify: `skills/research/SKILL.md`
- Modify: `skills/research/references/templates.md`

- [ ] **Step 1: Edit `skills/research/SKILL.md` — add Phase 1.5**

Insert a new phase between Phase 1 and Phase 2. Replace the existing Phase 1 and Phase 2 sections with:

```markdown
## Phase 1: Topic Research

1. Use `til_list(search=topic)` to check already-learned topics → avoid duplicate backlog entries
2. Research the topic via web search, identify required concepts, terms, and prerequisites
3. Break down into subtopics and research each directly
4. Analyze dependencies between subtopics

## Phase 1.5: Save Raw Sources

For each key source found during research:

1. Save to `./raw/{category}/{slug}.md` (auto-create folder)
2. Frontmatter format:
   ```yaml
   ---
   title: "Article Title"
   source: https://original-url
   fetched: YYYY-MM-DD
   tags:
     - {category}
     - {subtopic}
   ---
   ```
3. Body: the full content retrieved via WebFetch, converted to markdown
4. Raw files are immutable — if `raw/{category}/{slug}.md` already exists, skip (do not overwrite)
5. Record saved raw file paths for use in Phase 2

## Phase 2: Organize Backlog

1. Sort by learning order: Prerequisites → Core Concepts → Advanced
2. Add a one-line description per item
3. Gather user feedback: allow adding, removing, or reordering items
```

- [ ] **Step 2: Edit `skills/research/SKILL.md` — update Phase 3 sources reference**

In the Phase 3: Save section, after "Preserve existing sources, only add new items", add:

```markdown
   - sources values: use raw file paths (`raw/{category}/{slug}.md`) when saved in Phase 1.5, fall back to URLs for sources not saved locally
```

- [ ] **Step 3: Edit `skills/research/references/templates.md` — update backlog template sources**

Replace the existing sources example in the backlog template:

```yaml
sources:
  slug-a: [https://url-1]
```

With:

```yaml
sources:
  slug-a: [raw/{category}/source-article.md, https://url-1]
```

Update the field description:

```markdown
- `sources`: Map of slug → source list (raw file paths and/or URLs). Raw paths are preferred when the source was saved during /research. Used by `/til` skill to fetch reference material when learning a backlog item
```

- [ ] **Step 4: Commit**

```bash
git add skills/research/SKILL.md skills/research/references/templates.md
git commit -m "feat(research): save raw sources during research phase"
```

### Task 2: Update /til skill — read from raw/ first

**Files:**
- Modify: `skills/til/SKILL.md`

- [ ] **Step 1: Edit `skills/til/SKILL.md` — update Phase 1 source loading**

Replace the current source loading block (lines 21-23):

```markdown
   - When learning a backlog item, call `til_backlog_status`(category) → reference `sections[].items[].sourceUrls`
     - 1 URL: fetch directly with `WebFetch`
     - 2+ URLs: pass all URLs to a single `til-fetcher` subagent
```

With:

```markdown
   - When learning a backlog item, call `til_backlog_status`(category) → reference `sections[].items[].sourceUrls`
     - For each source entry:
       - If path starts with `raw/`: use `Read` to load local file (fast, reliable, no network)
       - If URL: fetch with `WebFetch`
     - 2+ sources (mixed raw + URL): read all raw files with `Read`, pass remaining URLs to a single `til-fetcher` subagent
```

- [ ] **Step 2: Commit**

```bash
git add skills/til/SKILL.md
git commit -m "feat(til): read from raw/ sources before falling back to WebFetch"
```
